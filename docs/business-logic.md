## 產品定位

### 🎯 核心價值主張
RenderResume 是一個採用 AI 技術的專業履歷與作品集生成器，基於 Fortune 500 企業標準的六維度評估模型，運用 STAR 原則架構，協助求職者打造具有競爭力的履歷。

### ✨ 核心特色
- **AI 智能分析**: 使用 GPT-4 進行深度履歷分析
- **專業評分**: Fortune 500 企業標準的六維度評估
- **STAR 重構**: 自動將經歷重構為有說服力的 STAR 框架
- **智能問答**: 互動式優化建議與即時履歷改善
- **多語言支援**: 繁體中文與英文介面

### 📊 目標用戶
- **求職者**: 需要專業履歷優化的個人用戶
- **學生**: 準備進入職場的畢業生
- **轉職者**: 需要重新包裝履歷的專業人士
- **企業**: 需要批量履歷分析的 HR 部門

## 六維度評分模型

RenderResume 採用基於 Fortune 500 企業標準的六維度評估模型，每個維度都有詳細的評分標準：

### 1. 💻 技術深度與廣度 (25%)

#### A+ 等級: 技術領域頂尖專家，引領技術趨勢
**評估標準**:
- 掌握前瞻性技術，引領技術趨勢
- 具備完整技術架構設計能力
- 在技術領域有深度創新和突破
- 具備跨技術棧的整合能力

**評分依據**: 技術棧廣度、架構設計經驗、創新技術應用

#### A 等級: 資深技術專家，具備完整技術棧與架構能力
**評估標準**:
- 精通多項核心技術
- 具備系統架構設計經驗
- 擁有豐富的技術選型經驗
- 能獨立解決複雜技術問題

#### A- 等級: 熟練技術人員，具備獨立開發與解決複雜問題能力
**評估標準**:
- 熟練掌握主要技術棧
- 能獨立完成複雜功能開發
- 具備良好的代碼品質意識
- 有解決技術難題的經驗

### 2. 🚀 項目複雜度與影響力 (25%)

#### A+ 等級: 主導企業級/產業級重大項目，具備顯著商業影響
**評估標準**:
- 主導企業級或產業級重大項目
- 項目具有顯著商業影響力
- 在項目中實現創新突破
- 具備可量化的重大成果

**評分依據**: STAR 原則完整性、量化成果、影響範圍

#### A 等級: 負責大型複雜項目，具備可量化的重大業務成果
**評估標準**:
- 負責大型複雜項目的核心開發
- 項目具有明確的業務價值
- 能量化展示項目成果
- 在團隊中發揮關鍵作用

#### A- 等級: 參與大型項目並擔任核心角色，有明確的重要技術貢獻
**評估標準**:
- 在大型項目中擔任重要角色
- 有明確的技術貢獻
- 解決項目中的關鍵技術問題
- 項目獲得良好效果

### 3. 💼 專業經驗完整度 (20%)

#### A+ 等級: 職涯軌跡完美，具備戰略思維與卓越領導經驗

## STAR 原則評估框架

系統採用 STAR 原則對項目經驗和工作成果進行評估：

### ⭐ STAR 原則詳解

#### S - Situation (情境) - 25%
- **識別項目背景**: 業務環境、技術挑戰、市場條件
- **評估問題複雜度**: 技術難度、業務影響、資源限制
- **分析約束條件**: 時間壓力、預算限制、團隊規模

#### T - Task (任務) - 25%
- **明確具體職責**: 個人承擔的具體任務和目標
- **評估任務難度**: 技術挑戰、業務複雜度、創新要求
- **分析角色匹配**: 任務與個人能力的匹配程度

#### A - Action (行動) - 25%
- **識別技術方案**: 採用的具體技術和解決方案
- **評估創新程度**: 解決方案的創新性和有效性
- **分析執行能力**: 決策過程和執行效率

#### R - Result (結果) - 25%
- **量化項目成果**: 具體的業務影響和技術成果
- **評估可持續性**: 技術成果的長期價值
- **分析團隊貢獻**: 對組織和團隊的具體貢獻

### 📊 STAR 評分權重

| 維度 | 權重 | 評估重點 |
|------|------|----------|
| **Situation** | 25% | 問題背景的複雜度和影響力 |
| **Task** | 25% | 任務的難度和重要性 |
| **Action** | 25% | 解決方案的創新性和執行力 |
| **Result** | 25% | 成果的量化程度和影響力 |

### 🔍 STAR 評估標準

#### 完整度評估 (25%)
- **S 完整性**: 是否清楚描述背景和挑戰
- **T 明確性**: 是否明確說明任務和目標
- **A 具體性**: 是否詳細描述行動和方案
- **R 量化性**: 是否有具體的成果數據

#### 具體性評估 (25%)
- **S 具體性**: 背景描述是否具體明確
- **T 具體性**: 任務描述是否具體可執行
- **A 具體性**: 行動描述是否具體可驗證
- **R 具體性**: 結果描述是否具體可量化

#### 量化程度評估 (25%)
- **S 量化**: 背景影響是否有量化指標
- **T 量化**: 任務目標是否有量化標準
- **A 量化**: 行動過程是否有量化記錄
- **R 量化**: 結果成果是否有量化數據

#### 影響力評估 (25%)
- **S 影響力**: 背景問題的業務影響程度
- **T 影響力**: 任務對業務的重要性
- **A 影響力**: 行動對解決問題的貢獻
- **R 影響力**: 結果對業務的實際影響

## 評分等級制度

採用 11 級等第制評分系統：

### 📈 評分等級詳解

| 等級 | 分數範圍 | 標準描述 | 表現特徵 |
|------|----------|----------|----------|
| **A+** | 95-100% | 卓越表現，業界頂尖水準 | 引領行業標準，具備突破性創新 |
| **A** | 90-94% | 優秀表現，高於平均水準 | 超越一般標準，具備優秀能力 |
| **A-** | 85-89% | 良好表現，穩定優質水準 | 穩定高於標準，具備良好能力 |
| **B+** | 80-84% | 滿意表現，高於基準要求 | 滿足高標準，具備滿意能力 |
| **B** | 75-79% | 合格表現，符合基準要求 | 達到基本標準，具備合格能力 |
| **B-** | 70-74% | 基本表現，接近基準要求 | 接近基本標準，需要提升 |
| **C+** | 60-69% | 待改進，低於基準要求 | 低於基本標準，需要改進 |
| **C** | 50-59% | 需改進，明顯不足 | 明顯不足，需要系統性提升 |
| **C-** | 40-49% | 急需改進，嚴重不足 | 嚴重不足，需要重新學習 |
| **F** | 0-39% | 不合格，完全不符合要求 | 完全不符合要求，不適合相關職位 |

### 🔍 評分標準應用

#### 強制 F 評分規則
- **技術深度與廣度**: 完全無法提取到任何技能、專案技術棧或工作中使用的技術
- **項目複雜度與影響力**: 完全無法提取到任何專案或項目經驗
- **專業經驗完整度**: 完全無法提取到任何工作經驗
- **教育背景**: 完全無法提取到任何教育資訊
- **成果與驗證**: 完全無法提取到任何成就、獎項或證書
- **整體專業形象**: 因為履歷內容嚴重不足無法評估

#### 評分推理要求
每個評分都必須包含完整的 Chain of Thought 推理過程：

```typescript
interface ScoreComment {
  reasoning: string;      // 【推理過程】
  finalScore: string;     // 【最終評分】
  suggestions: string;    // 【改進建議】
}
```

## AI 分析流程

### 🤖 AI 處理架構

#### 輸入處理
```typescript
interface AIAnalysisInput {
  // 文字內容
  resume?: string;           // 履歷文字
  additionalText?: string;   // 補充說明
  
  // 文件內容
  files?: File[];           // 上傳文件
  
  // 結構化資料
  education?: Education[];   // 教育背景
  experience?: Experience[]; // 工作經驗
  projects?: Project[];      // 專案經驗
  skills?: string;          // 技能專長
  personalInfo?: PersonalInfo; // 個人資訊
  links?: Links;            // 相關連結
}
```

#### 分析流程
1. **內容提取**: 從各種輸入源提取文字內容
2. **結構化處理**: 將內容組織成結構化格式
3. **AI 分析**: 使用 GPT-4 進行深度分析
4. **結果解析**: 解析 AI 回應為結構化數據
5. **評分計算**: 應用六維度評分算法
6. **建議生成**: 生成具體改善建議
