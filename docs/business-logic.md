## 產品定位

### 🎯 核心價值主張
RenderResume 是一個採用 AI 技術的專業履歷與作品集生成器，基於 Fortune 500 企業標準的六維度評估模型，運用 STAR 原則架構，協助求職者打造具有競爭力的履歷。

### ✨ 核心特色
- **AI 智能分析**: 使用 GPT-4 進行深度履歷分析
- **專業評分**: Fortune 500 企業標準的六維度評估
- **STAR 重構**: 自動將經歷重構為有說服力的 STAR 框架
- **智能問答**: 互動式優化建議與即時履歷改善
- **多語言支援**: 繁體中文與英文介面

### 📊 目標用戶
- **求職者**: 需要專業履歷優化的個人用戶
- **學生**: 準備進入職場的畢業生
- **轉職者**: 需要重新包裝履歷的專業人士
- **企業**: 需要批量履歷分析的 HR 部門

## 六維度評分模型

RenderResume 採用基於 Fortune 500 企業標準的六維度評估模型，每個維度都有詳細的評分標準：

### 1. 💻 技術深度與廣度 (25%)

#### A+ 等級: 技術領域頂尖專家，引領技術趨勢
**評估標準**:
- 掌握前瞻性技術，引領技術趨勢
- 具備完整技術架構設計能力
- 在技術領域有深度創新和突破
- 具備跨技術棧的整合能力

**評分依據**: 技術棧廣度、架構設計經驗、創新技術應用

#### A 等級: 資深技術專家，具備完整技術棧與架構能力
**評估標準**:
- 精通多項核心技術
- 具備系統架構設計經驗
- 擁有豐富的技術選型經驗
- 能獨立解決複雜技術問題

#### A- 等級: 熟練技術人員，具備獨立開發與解決複雜問題能力
**評估標準**:
- 熟練掌握主要技術棧
- 能獨立完成複雜功能開發
- 具備良好的代碼品質意識
- 有解決技術難題的經驗

### 2. 🚀 項目複雜度與影響力 (25%)

#### A+ 等級: 主導企業級/產業級重大項目，具備顯著商業影響
**評估標準**:
- 主導企業級或產業級重大項目
- 項目具有顯著商業影響力
- 在項目中實現創新突破
- 具備可量化的重大成果

**評分依據**: STAR 原則完整性、量化成果、影響範圍

#### A 等級: 負責大型複雜項目，具備可量化的重大業務成果
**評估標準**:
- 負責大型複雜項目的核心開發
- 項目具有明確的業務價值
- 能量化展示項目成果
- 在團隊中發揮關鍵作用

#### A- 等級: 參與大型項目並擔任核心角色，有明確的重要技術貢獻
**評估標準**:
- 在大型項目中擔任重要角色
- 有明確的技術貢獻
- 解決項目中的關鍵技術問題
- 項目獲得良好效果

### 3. 💼 專業經驗完整度 (20%)

#### A+ 等級: 職涯軌跡完美，具備戰略思維與卓越領導經驗
**評估標準**:
- 職涯發展軌跡完美連貫
- 具備戰略思維和遠見
- 有卓越的領導管理經驗
- 在業界有一定影響力

**評分依據**: 工作經歷完整性、職位成長軌跡、責任範圍

#### A 等級: 職業發展優秀，具備成熟管理經驗與專業深度
**評估標準**:
- 職業發展路徑清晰優秀
- 具備成熟的管理經驗
- 在專業領域有深度積累
- 有明確的職涯成長軌跡

#### A- 等級: 職業發展良好，具備穩定的專業成長軌跡與管理潛力
**評估標準**:
- 職業發展軌跡良好
- 有穩定的專業成長
- 展現管理潛力
- 專業能力持續提升

### 4. 🎓 教育背景與專業匹配度 (15%)

#### A+ 等級: 頂尖學府背景，專業與職涯完美匹配
**評估標準**:
- 頂尖學府教育背景
- 專業與職涯完美匹配
- 持續學習能力卓越
- 有優秀的學術成就

**評分依據**: 教育機構聲譽、專業匹配度、學習成果

#### A 等級: 優秀教育背景，專業相關性高，學習能力強
**評估標準**:
- 優秀的教育背景
- 專業與工作高度相關
- 展現強烈學習能力
- 有持續的知識更新

#### A- 等級: 良好教育背景，專業匹配度高，具備良好學習基礎
**評估標準**:
- 良好的教育背景
- 專業匹配度較高
- 具備良好學習基礎
- 能運用所學於工作

### 5. 🏆 成果與驗證 (10%)

#### A+ 等級: 具備業界認可的重大成就，有權威第三方驗證
**評估標準**:
- 業界認可的重大成就
- 有權威第三方驗證
- 影響力卓著
- 成果具有突破性

**評分依據**: 獎項證書、量化成果、業界影響力

#### A 等級: 具備顯著的專業成就，有明確的量化指標與外部認可
**評估標準**:
- 顯著的專業成就
- 明確的量化指標
- 獲得外部認可
- 成果可驗證

#### A- 等級: 具備重要的專業成果，有良好的量化數據支撐
**評估標準**:
- 重要的專業成果
- 良好的數據支撐
- 成果具體可見
- 有一定影響力

### 6. ✨ 整體專業形象 (5%)

#### A+ 等級: 履歷展現卓越的專業素養與強烈個人品牌
**評估標準**:
- 卓越的專業素養
- 強烈的個人品牌
- 表達完美清晰
- 履歷結構優秀

**評分依據**: 內容組織、語言表達、格式規範

#### A 等級: 履歷結構優秀，專業形象清晰，表達能力強
**評估標準**:
- 履歷結構優秀
- 專業形象清晰
- 表達能力強
- 內容組織良好

#### A- 等級: 履歷組織良好，專業表達清楚，整體形象佳
**評估標準**:
- 履歷組織良好

## STAR 原則評估框架

系統採用 STAR 原則對項目經驗和工作成果進行評估：

### ⭐ STAR 原則詳解

#### S - Situation (情境) - 25%
- **識別項目背景**: 業務環境、技術挑戰、市場條件
- **評估問題複雜度**: 技術難度、業務影響、資源限制
- **分析約束條件**: 時間壓力、預算限制、團隊規模

#### T - Task (任務) - 25%
- **明確具體職責**: 個人承擔的具體任務和目標
- **評估任務難度**: 技術挑戰、業務複雜度、創新要求
- **分析角色匹配**: 任務與個人能力的匹配程度

#### A - Action (行動) - 25%
- **識別技術方案**: 採用的具體技術和解決方案
- **評估創新程度**: 解決方案的創新性和有效性
- **分析執行能力**: 決策過程和執行效率

#### R - Result (結果) - 25%
- **量化項目成果**: 具體的業務影響和技術成果
- **評估可持續性**: 技術成果的長期價值
- **分析團隊貢獻**: 對組織和團隊的具體貢獻

### 📊 STAR 評分權重

| 維度 | 權重 | 評估重點 |
|------|------|----------|
| **Situation** | 25% | 問題背景的複雜度和影響力 |
| **Task** | 25% | 任務的難度和重要性 |
| **Action** | 25% | 解決方案的創新性和執行力 |
| **Result** | 25% | 成果的量化程度和影響力 |

### 🔍 STAR 評估標準

#### 完整度評估 (25%)
- **S 完整性**: 是否清楚描述背景和挑戰
- **T 明確性**: 是否明確說明任務和目標
- **A 具體性**: 是否詳細描述行動和方案
- **R 量化性**: 是否有具體的成果數據

#### 具體性評估 (25%)
- **S 具體性**: 背景描述是否具體明確
- **T 具體性**: 任務描述是否具體可執行
- **A 具體性**: 行動描述是否具體可驗證
- **R 具體性**: 結果描述是否具體可量化

#### 量化程度評估 (25%)
- **S 量化**: 背景影響是否有量化指標
- **T 量化**: 任務目標是否有量化標準
- **A 量化**: 行動過程是否有量化記錄
- **R 量化**: 結果成果是否有量化數據

#### 影響力評估 (25%)
- **S 影響力**: 背景問題的業務影響程度
- **T 影響力**: 任務對業務的重要性
- **A 影響力**: 行動對解決問題的貢獻
- **R 影響力**: 結果對業務的實際影響

## 評分等級制度

採用 11 級等第制評分系統：

### 📈 評分等級詳解

| 等級 | 分數範圍 | 標準描述 | 表現特徵 |
|------|----------|----------|----------|
| **A+** | 95-100% | 卓越表現，業界頂尖水準 | 引領行業標準，具備突破性創新 |
| **A** | 90-94% | 優秀表現，高於平均水準 | 超越一般標準，具備優秀能力 |
| **A-** | 85-89% | 良好表現，穩定優質水準 | 穩定高於標準，具備良好能力 |
| **B+** | 80-84% | 滿意表現，高於基準要求 | 滿足高標準，具備滿意能力 |
| **B** | 75-79% | 合格表現，符合基準要求 | 達到基本標準，具備合格能力 |
| **B-** | 70-74% | 基本表現，接近基準要求 | 接近基本標準，需要提升 |
| **C+** | 60-69% | 待改進，低於基準要求 | 低於基本標準，需要改進 |
| **C** | 50-59% | 需改進，明顯不足 | 明顯不足，需要系統性提升 |
| **C-** | 40-49% | 急需改進，嚴重不足 | 嚴重不足，需要重新學習 |
| **F** | 0-39% | 不合格，完全不符合要求 | 完全不符合要求，不適合相關職位 |

### 🔍 評分標準應用

#### 強制 F 評分規則
- **技術深度與廣度**: 完全無法提取到任何技能、專案技術棧或工作中使用的技術
- **項目複雜度與影響力**: 完全無法提取到任何專案或項目經驗
- **專業經驗完整度**: 完全無法提取到任何工作經驗
- **教育背景**: 完全無法提取到任何教育資訊
- **成果與驗證**: 完全無法提取到任何成就、獎項或證書
- **整體專業形象**: 因為履歷內容嚴重不足無法評估

#### 評分推理要求
每個評分都必須包含完整的 Chain of Thought 推理過程：

```typescript
interface ScoreComment {
  reasoning: string;      // 【推理過程】
  finalScore: string;     // 【最終評分】
  suggestions: string;    // 【改進建議】
}
```

#### 加權平均計算
```typescript
const weightedScore = (
  technicalScore * 0.25 +
  projectScore * 0.25 +
  experienceScore * 0.20 +
  educationScore * 0.15 +
  achievementScore * 0.10 +
  imageScore * 0.05

```

```typescript
function scoreToGrade(score: number): LetterGrade {
  if (score >= 95) return 'A+';

  if (score >= 60) return 'C+';

  return 'F';
}
```

## AI 分析流程

### 🤖 AI 處理架構

#### 輸入處理
```typescript
interface AIAnalysisInput {
  // 文字內容
  resume?: string;           // 履歷文字
  additionalText?: string;   // 補充說明
  
  // 文件內容
  files?: File[];           // 上傳文件
  
  // 結構化資料
  education?: Education[];   // 教育背景
  experience?: Experience[]; // 工作經驗
  projects?: Project[];      // 專案經驗
  skills?: string;          // 技能專長
  personalInfo?: PersonalInfo; // 個人資訊
  links?: Links;            // 相關連結
}
```

#### 分析流程
1. **內容提取**: 從各種輸入源提取文字內容
2. **結構化處理**: 將內容組織成結構化格式
3. **AI 分析**: 使用 GPT-4 進行深度分析
4. **結果解析**: 解析 AI 回應為結構化數據
5. **評分計算**: 應用六維度評分算法
6. **建議生成**: 生成具體改善建議

## Smart Chat 智能問答系統

Smart Chat 是 RenderResume 的核心互動功能，提供基於 AI 的履歷優化對話體驗。

### 🎯 系統目標

#### 核心使命
- **深度優化**: 基於 STAR 原則進行深度履歷優化
- **個性化指導**: 根據用戶背景提供客製化建議
- **即時整合**: 建議可立即應用到履歷編輯器
- **專業體驗**: 模擬專業履歷顧問的互動體驗

#### 業務價值
- **提升用戶參與度**: 互動式體驗增加用戶黏性
- **提高履歷品質**: 深度追問確保履歷完整性
- **降低使用門檻**: 引導式對話降低用戶認知負擔
- **建立專業形象**: 專業顧問角色提升產品可信度

### 🤖 AI 角色設計

#### Remo 博士 - 企鵝履歷顧問
- **角色定位**: 專業且親切的企鵝履歷顧問博士
- **專業背景**: 人力資源與職涯規劃博士學位
- **溝通風格**: 友善、鼓舞、略帶可愛語助詞
- **個性特質**: 平衡專業與親和力，避免過度賣萌

#### 個性化稱呼系統
```typescript
// 根據用戶資訊提供適當稱呼

  : '由於未獲得用戶姓名，可以用「你」或「夥伴」等親切稱呼。';
```

### 📋 對話流程設計

#### 六階段對話模型

1. **[STAGE:TOPIC_INTEREST]** 話題興趣確認
   - 確認用戶對新主題的興趣和需求
   - 不產生 excerpt，僅進行興趣確認

2. **[STAGE:TOPIC_OPEN]** 主題開啟
   - 正式進入主題討論
   - **必須產生 excerpt** 摘錄履歷段落
   - 簡要說明該段落重點

3. **[STAGE:FOLLOWUP]** 深入追問
   - 基於 STAR 原則逐步追問
   - 每次僅問一個明確問題
   - 不得重複履歷已有資訊

4. **[STAGE:INFO_CHECK]** 資訊補全檢查
   - 檢查是否已具備 STAR 所需資訊
   - 若已齊全則不可重複追問

5. **[STAGE:SUGGESTION]** 建議產生
   - 當資訊充分時產生結論性建議
   - 包含具體的 patchOps 操作
   - 進行六維度分類

6. **[STAGE:TOPIC_TRANSITION]** 話題切換
   - 總結前一話題並產生 suggestion
   - 插入 `<NEXT_TOPIC>` token
   - 開啟新話題引導

#### 話題切換規則
```typescript
// 強制流程：每次切換話題必須依序執行
1. 先用自然語言總結前一話題建議結論（切換時必須產生 suggestion）
2. 必須插入 <NEXT_TOPIC> token（此 token 絕不可遺漏）
3. token 後必須才開始新話題開頭引導並確認用戶興趣
4. 新話題必須產生 excerpt
```

### 🔄 Issues 狀態管理系統

#### 三種狀態類型
- **in_progress**: 進行中的問題（最高優先處理）
- **pending**: 待處理的問題
- **completed**: 已完成的問題

#### 狀態轉換規則
```typescript
// excerpt 觸發 in_progress
if (aiExcerpt && aiExcerpt.issue_id) {
  // 將其他 in_progress 關閉為 pending
  // 設定目標 issue 為 in_progress
}

// suggestion 觸發 completed
if (aiSuggestion) {
  // 關閉唯一 active 的 issue
  // 標記為 completed 並保存建議
}
```

#### 優先級管理
- **最高優先**: 只要存在 in_progress 狀態，嚴禁切換話題
- **狀態同步**: 確保全域僅有一個 active（in_progress）issue
- **自動排序**: 狀態變動的 issue 自動移到最前面

### 📝 結構化回應系統

#### ChatResponse 格式
```typescript
interface ChatResponse {
  message: string;                    // AI 回覆內容（3-4句話）
  suggestion?: {                     // 履歷優化建議
    title: string;                   // 建議標題
    description: string;             // 建議細節（包含所有時間、職責、數據等）
    category: string;                // 六維度分類之一
    patchOps?: PatchOp[];            // 具體修改操作
  };
  quickResponses: string[];          // 快速回覆選項（至少4個）
  excerpt?: {                       // 履歷段落摘錄
    title: string;                   // 段落標題（15字內）
    content: string;                 // 摘錄內容（中文，用 \n 換行）
    source: string;                  // 來源分類（工作經驗/專案經驗/技能專長/教育背景）
    issue_id?: string;               // 關聯的 issue ID
  };
}
```

#### PatchOps 操作系統
```typescript
interface PatchOp {
  op: 'set' | 'insert' | 'remove';
  path: string;                      // 履歷路徑（如 'experience[0].achievements[2]'）
  value?: string;                    // 新內容（set/insert 時使用）
  index?: number;                    // 插入/刪除位置（可選）
}
```

#### 操作範例
```typescript
// set: 替換整個段落
{ "op": "set", "path": "experience[0].achievements[2]", "value": "完整的新段落內容" }

// insert: 在陣列中插入新項目
{ "op": "insert", "path": "experience[0].achievements", "value": "新增列點", "index": 3 }

// remove: 刪除指定項目
{ "op": "remove", "path": "experience[0].achievements[2]" }
```
